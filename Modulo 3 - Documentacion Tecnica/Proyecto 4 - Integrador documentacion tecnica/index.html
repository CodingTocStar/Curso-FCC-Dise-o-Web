Tablas

Las tablas son estructuras utilizadas para organizar información en filas y columnas. Durante el proyecto Balance Sheet aprendí a utilizar todos los elementos semánticos involucrados.

Una tabla completa está formada por partes bien diferenciadas.

Aprender la semántica permite separar la estructura del estilo y facilita el acceso para tecnologías asistivas.

Dentro de thead van los encabezados, mientras que tbody contiene el contenido de la tabla.

Ejemplo de estructura básica:

<table>
  <caption>Título</caption>
  <thead>
    <tr><th>Columna</th></tr>
  </thead>
  <tbody>
    <tr><td>Dato</td></tr>
  </tbody>
</table>

Con esta estructura es posible crear documentos contables, registros e incluso dashboards.

  • table define la tabla.
  • caption asigna un título descriptivo.
  • thead contiene los encabezados.
  • tbody contiene los datos.
  • tr, th y td definen el contenido.

Selectores de atributo

Los selectores de atributo permiten aplicar estilos basados en valores presentes en atributos HTML.

En el proyecto del Balance Sheet utilicé el selector [class~="sr-only"] para mejorar accesibilidad.

Este selector identifica cualquier elemento cuyo atributo class contenga la palabra exacta.

Es una técnica esencial para interactuar con contenido invisible pero accesible.

span[class~="sr-only"] {
  clip-path: inset(50%);
  width: 1px;
}

Los selectores de atributo son muy útiles para estilos condicionales sin añadir clases adicionales.

  • [attr] selecciona elementos con ese atributo.
  • [attr="valor"] selecciona coincidencia exacta.
  • [class~="valor"] coincide con una palabra dentro de class.
  • Son ideales para accesibilidad.
  • Permiten un CSS más expresivo.

Pseudoelementos

Los pseudoelementos permiten generar contenido visual sin modificar el HTML real.

Durante el proyecto Responsive Piano los utilicé para crear las teclas negras sin añadir nodos extra.

Esto mantiene el código HTML limpio y mejora la mantenibilidad.

.key.black--key::after {
  content: "";
  position: absolute;
  width: 32px;
}

Se colocan respecto al contenedor cuando este tiene position: relative.

  • ::before inserta contenido al inicio.
  • ::after inserta contenido al final.
  • Son ideales para detalles decorativos.
  • Reducen la necesidad de HTML adicional.
  • Separan estilo de estructura.

Posicionamiento

El posicionamiento es clave para crear interfaces visualmente complejas.

En el piano, utilicé position: relative para definir un sistema de referencia para teclas negras generadas con pseudoelementos.

Y utilicé position: absolute para colocar elementos sobre otros.

.key {
  position: relative;
}

.key.black--key::after {
  position: absolute;
  top: 0;
}

También apliqué float en las teclas blancas para lograr una distribución horizontal simple.

  • relative crea un nuevo contexto de posicionamiento.
  • absolute saca el elemento del flujo.
  • sticky combina fixed + relative.
  • fixed mantiene algo siempre visible.
  • Son fundamentales para layouts avanzados.

Responsive Design

Gracias a las media queries pude adaptar el diseño del piano a diferentes tamaños de pantalla.

Las media queries activan reglas solo cuando las condiciones se cumplen.

@media (max-width: 768px) {
  #piano { width: 358px; }
}

Esto permite crear interfaces flexibles y accesibles.

  • max-width se activa en pantallas pequeñas.
  • min-width organiza reglas para pantallas grandes.
  • Permiten rediseñar layouts.
  • Evitan desbordes.
  • Son esenciales para cualquier web moderna.
